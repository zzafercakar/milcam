## Technical Insights

### Why File-Based Backdrop Works
- **Problem:** Live capture at startup is fragile — if capture happens during system chaos, bad frame locks in
- **Solution:** One-time capture from clean boot, saved to file. Then code loads from file, never "corrupts" again
- **Scalability:** New units → one bootstrap capture → copy to file → forever clean

### Signal Handling for Graceful Exit
- Qt's normal quit path (`QApplication::quit()`) runs destructors and cleanup slots
- SIGTERM handler sets a flag; a QTimer detects it and triggers the actual cleanup
- **Why timer?** Signal handlers must be signal-safe (limited API). Timer runs in main event loop (full Qt available)
- **Result:** `killall -TERM` now properly restores CNC frame before exit (no frozen MilCAM pixels)

### Architecture Pattern: Last-Writer Wins
- Single `/dev/fb0` device shared by all processes
- Whoever writes fb0 last is what's visible (regardless of VT state)
- CodeSYS (tty1) and MilCAM (tty2/default) both paint directly to fb0
- No window manager, no X11 — just raw frame buffer

## Key Learnings

1. **Graceful degradation > heroic fixes** — saved backdrop is once-per-device setup; live capture is always fallback
2. **Signal handlers need event loop integration** — SIGTERM sets flag, Qt timer checks it (not direct cleanup)
3. **Frame buffer is "last writer wins"** — no VT switching needed; app that writes fb0 last is visible
4. **Device tests beat unit tests for UI** — screenshots on panel are more trustworthy than host emulation
